# Get started with zrok

zrok lets you share a running local service—or a directory of files—securely over the internet without opening
firewall ports or configuring infrastructure. This guide takes you from zero to your first working share, then shows
you how to keep shares running reliably with the zrok agent.

## Choose your deployment

Before you begin, decide how you'll use zrok:

- **myzrok.io (hosted)** — NetFoundry runs the infrastructure. Sign up for a free account at
  [myzrok.io](https://myzrok.io) and skip the deployment step entirely.
- **Self-hosted** — You run your own zrok instance on [Linux](../self-hosting/deployment/linux/),
  [Docker](../self-hosting/deployment/docker), or [Kubernetes](../self-hosting/deployment/kubernetes). Complete your
  deployment first, then return here with your instance's API endpoint URL.

Both paths converge at Step 2 (installing the `zrok2` command).
